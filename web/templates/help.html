本机 IP 来自默认网卡。如果电脑有多张网卡，请确认手机和电脑在同一 Wi-Fi。

前置条件

  • 手机和电脑连同一个 Wi-Fi（手机热点也可，电脑连手机热点更稳）。
  • 电脑防火墙允许 {{ proxy_port }} 端口入站（macOS 默认允许局域网；Windows 首次会弹确认框）。
  • 已经 python main.py 启动，控制台没有报错。

步骤 1 · 设置手机 Wi-Fi 代理

  1. 打开「设置 → Wi-Fi」，长按当前已连接的 Wi-Fi，点「修改网络」。
  2. 展开「高级选项」，「代理」改为 手动。
  3. 代理主机名填 {{ lan_ip }}，端口填 {{ proxy_port }}，保存。

不同厂商位置略有差异：MIUI/HyperOS、ColorOS、OneUI 都在 Wi-Fi 详情里，搜「代理」即可。

步骤 2 · 安装 mitmproxy 根证书（HTTPS 必装）

  1. 手机浏览器打开 http://mitm.it，能看到下载页才说明代理通了。
  2. 点 Android 那一栏下载 mitmproxy-ca-cert.cer。
  3. 打开「设置 → 安全 → 加密与凭据 → 安装证书 → CA 证书」，选择刚下载的文件。
  4. 取个名字保存即可。

如果 mitm.it 显示「If you can see this, traffic is not passing through mitmproxy」，说明代理没生效，回到步骤 1 检查 IP 和端口。

步骤 3 · Android 7+ 抓 App HTTPS

从 Android 7 起，App 默认不信任用户安装的 CA 证书，只有以下情况能抓到：

  • App 自己声明信任用户证书（在 network_security_config.xml 加 <certificates src="user"/>）。自家 App 改一下重新打包即可。
  • 系统证书：root 过的机器（或模拟器）把证书放到 /system/etc/security/cacerts/。
  • 用 frida / objection 在运行时绕过 SSL Pinning。

浏览器流量不受影响，装好用户证书就能抓。

验证

  1. 手机访问任意 HTTP 站点，比如 http://example.com。
  2. 回到本页左侧菜单的「流量」页，应该立刻看到对应请求。
  3. 在「规则」页新建一条规则，URL pattern 用 https://example.com/*，再访问就会被 mock。

常见问题

手机打不开任何网页
代理 IP 或端口写错；或电脑没开 mock-server；或防火墙挡了 {{ proxy_port }} 端口。
mitm.it 打不开
代理没生效。确认手机 Wi-Fi 详情里代理确实是「手动」且 IP={{ lan_ip }} 端口={{ proxy_port }}。
HTTP 能抓，HTTPS 全是错
根证书没装或没被信任。重做步骤 2，注意要安装到「CA 证书」而不是「VPN 和应用证书」。
App 抓不到（浏览器能抓）
该 App 不信任用户证书，参考步骤 3。
电脑 IP 总是变
给电脑配静态 IP，或在路由器后台给 MAC 地址绑定固定 IP。
